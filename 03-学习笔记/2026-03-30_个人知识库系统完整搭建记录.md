# 个人知识库系统完整搭建记录

## 背景

在 AI 时代，个人需要构建自己的知识库系统。传统网页端 AI 对话存在数据沉淀难题，本地个人知识库系统能解决这一问题。

## 核心组件

| 组件 | 工具 | 作用 |
|------|------|------|
| 存储+显示 | Obsidian | 本地笔记软件，开源免费 |
| AI 交互层 | OpenClaw/QClaw | 与知识库对话 |
| 云端同步 | GitHub | 免费云端备份 |

## 搭建过程

### 1. 安装 Obsidian

- 版本：v1.12.7
- 安装路径：`C:\Program Files\Obsidian\`
- 下载方式：通过 ghfast.top 镜像下载（GitHub 直连超时）

### 2. 创建本地仓库

- 仓库路径：`C:\Users\Tom\knowledge-vault`
- 目录结构：
  ```
  00-日记/         ← 每日日记
  01-工作/         ← 工作笔记
  02-AI资讯/       ← AI 行业动态
  03-学习笔记/     ← 学习沉淀
  04-项目/         ← 项目记录
  05-资源库/       ← 收藏素材
  06-归档/         ← 已完成内容
  99-草稿箱/       ← 临时文件
  ```

### 3. 配置 GitHub 同步

- 远程仓库：`github.com/xiaodoudoudabaozi/knowledge-vault`
- 认证方式：GitHub CLI (`gh auth login`)
- 问题：Git 直连 GitHub 超时，需用 Token 配合推送

### 4. 安装 Obsidian 插件

通过 Obsidian 社区插件市场安装：
- ✅ QuickAdd — 快速创建笔记
- ✅ Templater — 强大的模板系统

### 5. 创建 Skills

| Skill | 功能 |
|-------|------|
| session-archiver | 会话内容自动归档 |
| ai-news-feed | AI 资讯定时抓取 |
| diary-behaviour | 日记行为分析 |

### 6. 配置定时任务

| 任务 | 时间 | 说明 |
|------|------|------|
| AI日报 | 每天 09:00 | 抓取 AI 新闻存知识库 |
| 周报 | 每周日 21:00 | 分析日记生成总结 |

## 遇到的问题与解决

### GitHub 直连超时
- **问题**：`git push` 连接 GitHub 超时（TCP 443）
- **解决**：使用 ghfast.top 镜像 + GitHub Token 认证

### GitHub Push Protection 拦截
- **问题**：脚本中包含 Token 被 GitHub 安全策略拦截
- **解决**：删除含 Token 的脚本，重写 Git 历史

### 插件下载失败
- **问题**：网络问题无法从 GitHub 下载插件
- **解决**：通过 Obsidian 内置插件市场安装

## 使用指南

### 每天写日记
1. 打开 Obsidian
2. `Ctrl + N` 新建笔记
3. 保存到 `00-日记/`

### 查看AI日报
- 每天 09:00 自动生成到 `02-AI资讯/`
- 或直接跟我说「抓一下 AI 新闻」

### 归档对话
- 说「存一下」或「归档」
- 我会整理保存到对应目录

### 同步 GitHub
- 定时任务自动推送
- 或手动运行 git push

## 关键要点

1. **知识库的价值在于持续沉淀**，不是一次性整理
2. **定时任务自动化后维护成本很低**
3. **GitHub Token 不要硬编码在脚本里**
4. **Obsidian 插件可以极大提升效率**
